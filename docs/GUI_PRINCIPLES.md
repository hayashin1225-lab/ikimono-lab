# GUI共通原則

## 主役と情報階層

- 生き物を画面の主役とし、生き物名、画像、レア度を最初の認識単位とする。
- 研究資料は重要だが最初の視線を無条件に奪わせない。
- 主要操作を目立たせるために内容表示を過度に圧迫しない。

## 操作

- 子どもが次の操作を迷わず、必要な瞬間に主要操作を発見できるようにする。
- 主要操作を他要素に隠さず、戻る・閉じる・続けるの意味を明確にし、タップ対象を小さくしすぎない。

## 情報量

- 情報量の問題を単純な削除だけで解決しない。必要に応じて段階的に開示する。
- ゲームらしさと研究資料らしさを排他的に扱わず、子ども向けでも情報を空疎にしない。

## レスポンシブ

- スマートフォンを主要基準とし、文字・ボタンの重なりや意図しない横スクロールを許容しない。
- 画面幅だけでなく画面高、セーフエリアを確認する。
- 固定表示は占有面積に見合う価値がある場合に限る。

## 実装と検収

- Figma上の成立だけで完了とせず、実ブラウザで対象状態へ到達して確認する。
- 静的HTMLを開いただけ、スクリーンショットだけで操作性確認完了としない。
- GUI改善を理由にゲームロジックや保存形式を無断変更しない。
- 公開版を確認するまで完全完了としない。

## 原則変更

原則は固定的な絶対解ではない。変更理由は[DECISIONS.md](DECISIONS.md)へ記録し、成功例と失敗例はGUIカルテ（案件ごとのGUI仕様・レビュー記録）へ反映する。

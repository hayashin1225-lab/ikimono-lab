# AIオーケストレーション.runtime

## 定義

AIオーケストレーション.runtimeは、複数のAI、人間、ツール、成果物からなるAIオーケストレーションで、人間が担ってきた情報伝達、実行順序、状態管理、排他、停止、成果物回収、証跡記録を実行統制基盤へ外在化する運用形態である。

短縮すると、人間がAI群のランタイムである状態から、専用のランタイムを持つ状態への移行である。

`.runtime`は無印のAIオーケストレーションを否定せず、その上位段階や成熟度を意味しない。探索的作業、機密業務、高リスク業務では、人間中心の運用を維持できる。

## 意味判断と実行統制

人間は製品ビジョン、意味、感覚、優先順位、リスク受容、マージ・公開の最終判断を担う。ChatGPTは要求整理、仕様化、実行許可、PR監査を担う。連絡将校は承認済みの条件を機械的に判定し、Codexの起動、Git・GitHub操作、状態・ログ管理を担う。Codexは調査、最小差分実装、検査、機械可読な報告を担う。

連絡将校はIssue本文の意味を解釈せず、本文やコメント中のコマンドを直接実行しない。承認済み設定と固定安全規則だけを実行する。

## MVPの位置づけ

初期実装はWindows PowerShell 5.1以上で動く「連絡将校MVP」である。単一PC・単一リポジトリ・一件処理を対象とし、GitHub Issueを共有状態、ローカルログを詳細証跡とする。

MVPは自動再試行、分散ロック、複数PC・複数リポジトリ処理、他AIアダプター、GitHub Actions、Figma API操作を含まない。自動マージ、自動公開、自動承認を行わない。

## 将来拡張

将来の拡張は、人間の承認、権限分離、共有成果物、独立査読を保ったまま段階的に評価する。Claude、Gemini等を含む特定製品・モデルの採用は、このMVPでは決定・実装していない。

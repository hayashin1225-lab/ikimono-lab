# 短編共通基盤・必須手動QA

## 実行前

1. 商品フォルダーをローカルへコピーします。
2. ネットワークを切断します。
3. `index.html` をダブルクリックし、EdgeまたはChromeで開きます。
4. 開発者ツールのConsoleを開きます。

## 1366×768と1920×1080で共通確認

- [ ] タイトル、拠点、行動条件、結果、図鑑、エンディング、設定・記録の7画面へ到達できます。
- [ ] 主要ボタンが画面外へ隠れず、キーボードのTab操作でもフォーカスできます。
- [ ] 長いタイトルと長い標本名を入れても、文字やボタンが重なりません。
- [ ] 条件A・Bの両方から結果画面へ進めます。
- [ ] 2種類の標本を集めるとエンディングが解放されます。
- [ ] 図鑑に標本名、説明、記録回数が表示されます。
- [ ] Consoleに未処理例外がありません。

## 保存・再開

- [ ] 調査確定後にブラウザを閉じ、同じ `index.html` を再度開くと結果または安全な拠点から再開できます。
- [ ] 音量と「動きを抑える」を変更し、再起動後も設定が維持されます。
- [ ] 記録JSONを書き出せます。
- [ ] 同じ記録JSONを2回読み込んでも、標本種類数・回数・行動回数が2回目に増えません。
- [ ] 現在より古い記録JSONを読み込んでも進捗が後退しません。
- [ ] 別の `titleId` を持つJSONはエラー表示となり、現在の記録は変わりません。

## 破損データ復旧

1. 開発者ツールで、作品の保存キーへ不正な文字列を設定します。
2. ページを再読込します。

- [ ] 自動で不正データを上書きしません。
- [ ] 復旧カードが表示され、元データをテキストとして退避できます。
- [ ] 「初期状態で再開」で新しい有効な保存データを作れます。
- [ ] 「保存せず確認する」で、元データを残したまま一時利用できます。

## 商品フォルダー確認

- [ ] ZIP展開後の相対パスが維持されています。
- [ ] `index.html` の直開きで起動します。
- [ ] サーバー、アカウント、インストーラー、管理者権限を要求しません。
- [ ] ネットワークパネルに実行時通信がありません。
- [ ] Windows 10／11のEdge・Chrome現行安定版で最後まで完走できます。

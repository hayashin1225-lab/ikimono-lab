# 自然史遊戯録・短編共通基盤

Issue #24で構築する、3作品パイロット用の最小共通基盤です。既存の「親子で育てるいきもの研究所」本体とは別の製品フォルダーとして配置しています。

## 起動

`index.html` をEdgeまたはChromeで直接開きます。Webサーバー、インストール、アカウント、ネットワーク接続は不要です。

## 構成

| パス | 責任 |
|---|---|
| `index.html` | 共通7画面のHTML |
| `assets/css/app.css` | 共通表示 |
| `assets/js/core.js` | 保存、復旧、画面状態、記録統合の純粋処理 |
| `assets/js/app.js` | DOM操作とブラウザー入出力 |
| `data/game-data.js` | 作品名、条件、標本、結末、結果決定 |
| `qa/` | 自動テストと必須手動QA |
| `store/` | 商品説明、画像、ZIP、提出チェックの雛形 |

`core.js` は作品固有名を持ちません。各作品では原則として `data/game-data.js` と視覚素材を差し替え、固有の中心ルールが必要な場合だけ、最小範囲で `resolveResult` とUIを拡張します。

## セーブと記録

- 主要行動確定後に `localStorage` へ自動保存します。
- 保存キーは作品ごとに `data/game-data.js` の `meta.storageKey` で分離します。
- 破損データは自動上書きせず、退避、初期化、保存なし利用の導線を表示します。
- 記録JSONは同一 `titleId` のみ統合します。
- 標本回数と行動回数は大きい値を維持し、実績は和集合にするため、同一JSONの再読込で重複や後退が起きません。

## 自動テスト

Node.js 20以降で次を実行します。

```powershell
powershell -ExecutionPolicy Bypass -File .\qa\run-tests.ps1
```

または、製品フォルダーで次を実行します。

```text
node --test qa/*.test.js
```

画面寸法、Windowsブラウザー、オフライン完走は [`qa/MANUAL_QA.md`](qa/MANUAL_QA.md) に従って実機確認します。

## 作品化の最小手順

1. このフォルダーを新しい商品IDのフォルダーへ複製します。
2. `meta.titleId` と `meta.storageKey` を、既存作品と重複しない値へ変更します。
3. 条件、標本、結果決定、エンディングを作品仕様へ差し替えます。
4. 仮標本が残っていないことを自動テストへ追加します。
5. `store/` の提出物を埋めます。
6. 自動テストと必須手動QAを完了します。

将来使う可能性だけを理由に新しい抽象層は追加しません。第2作、第3作で実際に重複した処理だけを共通基盤へ戻します。
